PEET testing structure


  • All test parameters should be stored in a matlab file

    • create a template generator

  • That should be an argument to the

    • volume generator init functions

    • test / simulation driver

  • Parameters to specify

    • testName

    • volume size

    • type of particle

      • step-ramp-sine

      • random

      • geometric structures

      • biological approximations

    • number of particles in the volume

      • position of each particle or way it is chosen

      • orientation of each particle or way it is chosen

    • particle volume extent

    • CCD/intensity offset

    • noise parameters

      • distribution

      • distribution parameters

        • possibly specified in SNR

      • noise generation should be modular so it can be added later as well

    • missing wedge range

  • Output files

    • MRC volume

    • particle location model

    • true alignment structure

    • reference volume

  • Tests to run on the simulated volume

    • Validation

      • alignment structure modifications

      • shift sign sense (document)

      • rotation order /direction (document)

      • Averaging of particles using motlAverage*

    • Performance

      • full particle search and average

      • particle search and compare alignment structures





Running a simulation

  1. Create a test parameter data (matlab) file

    1. copy, modify and execute the simParticleVol_Template.m file

    2. the test parameter data file is named testName.mat

  2. Create the simulated volume, particle model and alignment structure using

    1. genSimParticleVol(‘testName’)

  3. Run the test on the simulated volume
